# Overview

This release expands the platform's RSS ingestion coverage, removes collector onboarding bottlenecks, and introduces a scalable data-driven collector architecture.

The release also includes extensive runtime validation, feed discovery research, operational audits, and production verification.

---

# Major Changes

## 1. Data-Driven Collector Factory

### Previous Architecture

Collectors were instantiated using source-specific conditional logic.

Example:

```python
if slug == "livemint-markets":
    return LiveMintRSSCollector(...)
elif slug == "et-markets":
    return EconomicTimesRSSCollector(...)
```

Adding new feeds required:

* Creating new collector classes
* Modifying orchestration code
* Updating deployment artifacts

### New Architecture

Collectors are now instantiated dynamically from source metadata.

```python
if meta.source_type == "rss":
    return BaseRSSCollector(meta)
```

Benefits:

* Zero-code RSS onboarding
* Reduced maintenance burden
* Simplified orchestration layer
* Improved scalability
